# Data Model: 优化博客 UI 与发布体验

## 1. 文章（Post）

- **标识**：
  - `id`: 文章唯一标识（已有）
  - `slug`: URL 友好标识，用于详情页路由（已有，通过迁移保证存在）
- **核心内容**：
  - `title`: 标题（必填，写作页前后端均需校验非空）
  - `content`: 正文/markdown 内容（必填，写作页前后端均需校验非空）
- **作者与时间**：
  - `authorId`: 作者用户 id（用于「我的文章」过滤）
  - `createdAt`: 创建时间
  - `updatedAt`: 最后更新时间
- **展示相关**：
  - 首页列表展示依赖 `title`、`slug`（或 `id`）、部分摘要/时间信息。

### 验证规则

- `title`：
  - 非空，去除前后空白后长度 > 0。
  - 可在实现阶段约束最大长度，避免极端长标题影响布局。
- `content`：
  - 非空，去除前后空白后长度 > 0。
  - 支持 markdown 语法，其具体渲染由前端 markdown 组件负责。

### 状态与变更

- **创建**：写作页提交合法 `title` 与 `content` 后，经后端校验成功写入。
- **发布后可见性**：发布成功后必须能通过首页数据源立即读取到新记录，以便列表展示。

## 2. 用户（User）

- **标识与基础信息**：
  - `id`: 用户唯一标识。
  - `name` / `displayName`: 展示名，用于头像占位与菜单文案。
- **头像信息**：
  - `avatarUrl`: 可选头像链接。
  - 若 `avatarUrl` 为空，则根据 `displayName` 首字母或其他规则生成占位头像（例如字母圆形背景）。

### 会话与视图

- 登录状态下：
  - 首页顶部右侧显示头像组件。
  - 头像下拉菜单展示「我的文章」「退出」两个操作项。
- 未登录状态下：
  - 不显示头像入口，也不展示个人菜单。

## 3. 视图与派生模型（非持久化）

- **首页文章列表视图**：
  - 衍生自 `Post`，包含首页展示所需字段（标题、时间、摘要/前几行内容、slug）。
  - 用于支持发布成功后跳转回首页并在列表中找到新文章。

- **「我的文章」列表视图**：
  - 衍生自 `Post`，在数据层通过 `authorId == 当前用户 id` 进行过滤。
  - 用于头像菜单中的「我的文章」入口，对应单独的路由/页面。
