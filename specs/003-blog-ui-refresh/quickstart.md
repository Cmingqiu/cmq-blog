# Quickstart: 优化博客 UI 与发布体验

## 1. 环境准备

1. 安装依赖：

   ```bash
   npm install
   ```

2. 初始化/迁移本地 SQLite 数据库（如尚未执行）：

   ```bash
   npm run db:migrate
   ```

3. 启动开发服务：

   ```bash
   npm run dev
   ```

## 2. 功能验证路径

### 2.1 发布文章后跳转首页

1. 在浏览器打开本地站点（例如 `http://localhost:3000`），完成登录。
2. 通过入口进入写作页面，填写合法的标题与正文。
3. 点击发布按钮：
   - 期望行为：发布成功后自动跳转到首页。
   - 验证点：新文章出现在首页文章列表靠前位置，点击可进入详情页。

### 2.2 错误与表单反馈

1. 在写作页清空标题或正文，尝试直接点击发布。
   - 期望行为：前端阻止提交，并在相应字段附近显示必填提示文案。
2. 临时模拟接口错误（如后端返回 500 或人为抛错），再次点击发布。
   - 期望行为：页面在显眼位置提示「发布失败」类错误信息，而不是静默失败。
3. 在网络断开或较差的环境下发起发布请求。
   - 期望行为：页面提示网络相关错误，并允许用户重试或返回首页。

### 2.3 写作页面宽度与预览一致性

1. 在桌面端宽屏浏览器中打开写作页面。
2. 对比：
   - 写作页编辑/预览区域宽度是否接近阅读页内容宽度，不再出现明显的窄列。
   - 预览与阅读页在段落、标题、列表等基础排版上的样式是否基本一致。
3. 确认顶部提示文案已被返回首页的箭头按钮替换，点击后能可靠返回首页。

### 2.4 首页 UI 与绿色系品牌

1. 打开首页，观察整体布局：
   - 线条风格装饰与「Markdown 写作、标签分类、搜索、评论与主题切换」等入口已被移除。
   - 内容列表成为视觉主焦点。
2. 检查品牌元素：
   - 顶部站点标识已替换为绿色系 logo（非纯文字「CMQ Blog」）。
   - 浏览器标签中显示的 favicon 与该 logo 风格一致。
   - 页面整体仍保持中性色调，未大量使用绿色作为主色。

### 2.5 登录头像下拉菜单

1. 使用已存在的账号登录后访问首页。
2. 观察首页顶部右侧是否展示用户头像（或首字母占位头像）。
3. 将鼠标悬停或点击头像：
   - 期望行为：出现包含「我的文章」「退出」两个入口的下拉浮层。
4. 点击「我的文章」：
   - 期望行为：跳转至仅展示当前登录用户文章列表的页面。
5. 回到首页头像下拉菜单，点击「退出」：
   - 期望行为：当前会话被清除，页面视图更新为未登录状态，头像与个人菜单不再显示。

## 3. 测试与回归

1. 运行自动化测试（如项目已有命令）：

   ```bash
   npm test
   ```

2. 确认与本 Feature 相关的新增/修改测试全部通过：
   - 文章发布成功流程与首页列表展示。
   - 写作页表单校验与错误提示。
   - 首页 UI 结构调整（可通过快照或 DOM 断言）。
   - 登录头像与下拉菜单行为。

3. 在浏览器开发者工具中观察首屏加载时间，确保首页与写作页首屏加载保持在宪章要求范围内（约 \(<= 3s\)）。
