# Quickstart: Personal Blog CMS (Acceptance + Testing)

**Branch**: `001-blog-cms`  
**Created**: 2026-03-12  
**Purpose**: 用可重复步骤验证每个用户故事的“独立可测试”与关键门禁（单元测试、首屏性能、简洁 UX）。

## Preconditions

- 已配置本地开发环境与必要的环境变量（在实现阶段补齐）
- 已有至少 1 个“博主/管理员”账号可登录

## User Story 1 (P1) - 写作：草稿/预览/发布

### Acceptance

1. 打开写作页 `/editor/new`
2. 输入标题与 Markdown 正文（含代码块/表格等），点击“预览”
3. 验证预览渲染正确；返回编辑后内容不丢失
4. 点击“保存草稿”，刷新页面后仍可加载草稿继续编辑
5. 点击“发布”，跳转或提示成功
6. 返回首页 `/` 并刷新，确认新文章出现在列表顶部
7. 进入文章详情页，确认正文展示与预览一致

### Testing (unit-first)

- 针对“发布校验（标题/正文必填）”、“Markdown 渲染管线一致性”、“发布后列表可见性更新”的核心逻辑必须有单元测试。

## User Story 2 (P2) - 标签分类与搜索

### Acceptance

1. 在首页查看文章列表与标签入口
2. 点击某个标签，列表仅显示包含该标签的文章；清除筛选后恢复完整列表
3. 输入关键词进行搜索，结果仅包含匹配文章；无匹配时显示明确空状态
4. 从搜索结果进入详情页，返回后搜索条件与结果仍保持

### Testing (unit-first)

- 搜索查询解析、筛选组合逻辑、空结果状态判定必须有单元测试。

## User Story 3 (P3) - 评论：登录后发表评论

### Acceptance

1. 以未登录状态打开文章详情页
2. 尝试发表评论，系统提示“需要登录”并提供入口
3. 使用 GitHub 登录后返回文章详情页
4. 输入评论并提交，评论立即出现在列表中并显示作者信息
5. 刷新页面，评论仍可见且顺序一致

### Testing (unit-first)

- “未登录禁止评论”、“评论正文必填校验”、“评论持久化与读取排序”核心逻辑必须有单元测试。

## User Story 4 (P3) - 明暗主题切换

### Acceptance

1. 在任意页面切换到暗黑主题
2. 刷新页面或重新打开站点，主题偏好保持
3. 验证文章内容（代码块/引用/图片）在暗黑主题下仍可读

### Testing (unit-first)

- “主题偏好持久化与读取”必须有单元测试（不依赖浏览器真实存储实现时可做抽象与模拟）。

## Constitution Gates (must remain true)

- 单元测试：本功能所有变更必须新增/更新单元测试并通过
- 性能：关键页面首屏 \(<= 3s\) 作为验收门禁（实现阶段补齐具体测量方式与阈值）
- UX：错误提示必须可理解并指向下一步
